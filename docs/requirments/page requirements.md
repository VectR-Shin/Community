# Layouts
### Main Layout
```
목적
- 전역 네비게이션(Header) 기능 제공
- 게시글 통합 검색 기능 제공
- 게시판 목록 접근 기능 제공
- 사용자 인증 상태에 따른 사용자 기능 제공 (내 프로필, 로그인, 로그아웃)

적용 대상
- 추가 정보 입력 페이지 이외의 모든 페이지

공통 구성
- Header
  - 로고
  - 통합 검색
  - 게시판 목록 버튼
  - 로그인 상태에 따른 UI
    - 로그인: 내 프로필 버튼, 로그아웃 버튼
    - 비로그인: 로그인 버튼

공통 필요 데이터
- 로그인한 사용자의 프로필 정보

공통 사용자 동작
- 메인 페이지 이동 (로고 클릭)
- 게시판 목록 페이지 이동
- 게시글 통합 검색
- 인증 상태에 기반 동작
  - 로그인 사용자
    - 내 프로필 페이지 이동
    - 로그아웃
  - 로그아웃 사용자
    - 로그인 (GET /oauth2/authorization/keycloak)

공통 호출 API
- 로그인한 사용자 정보 조회
  - GET /users/me

비고
- Header 렌더링 시 최초 1회 호출
```

# Pages
### 추가 정보 입력 페이지 (Onboarding)
```
목적
- 처음 접속한 사용자의 세부 정보 입력을 위한 페이지

접근 대상
- 최초로 로그인한 온보딩 미완료 USER

레이아웃
- Onboarding Layout

필요 데이터
(없음)

사용자 동작
- 닉네임 입력 [필수]
- 가입 완료

호출 API
- 사용자 세부 정보(닉네임 등) 제출
  - POST /users/onboarding
```

### 메인 페이지
```
목적
- 사이트의 메인 페이지
- 전체 인기글 목록을 제공
- 제목 혹은 내용 기준으로 전체 인기글을 검색하는 기능 제공

접근 대상
- 모든 사용자

레이아웃
- Main Layout

필요 데이터
- 인기글 목록

사용자 동작
- 인기글 목록 조회
- 인기글 세부 페이지 이동
- 제목 기준으로 인기글 검색
- 내용 기준으로 인기글 검색

호출 API
- 전체 인기글 목록 조회
  - GET /posts/popular?page={pageNumber}
- 전체 인기글 검색 (제목/내용 필터 포함)
  - GET /posts/popular?keyword={keyword}&searchType={TITLE | CONTENT}&page={pageNumber}

비고
- 인기글 목록은 20개씩 페이지네이션 되어서 제공한다.
- 인기글 선정 기준은 운영 정책에 따라 변경될 수 있다. 요구사항 문서를 참고
```

### 게시판 목록 페이지
```
목적
- 전체 게시판 목록을 카테고리별로 묶어서 한 번에 제공
- 선택한 카테고리에 해당하는 게시판 목록을 제공
- 게시판 이름 기반의 게시판 검색 기능 제공

접근 대상
- 모든 사용자

레이아웃
- Main Layout

필요 데이터
- 게시판 목록

사용자 동작
- 게시판 탐색
  - 전체 게시판 목록 조회 (카테고리별로 grouping)
  - 특정 카테고리를 선택하여 게시판 필터링
- 게시판 검색
  - 게시판 이름 기반 검색

호출 API
- 전체 게시판 목록 조회
  - GET /boards
- 게시판 이름 기반의 게시판 검색 결과 조회
  - GET /boards?keyword={keyword}&page={pageNumber}

비고
- 초기 화면에서는 전체 게시판 목록을 카테고리별로 묶어서 보여준다.
- 게시판 검색 결과는 20개씩 페이지네이션해서 제공한다.
- 게시판 검색 결과는 완전 일치, 부분 일치를 모두 허용하지만 완전 일치가 최상단에 위치한다.
- 카테고리 선택에 따른 게시판 목록은 GET /boards로 조회한 데이터를 프론트엔드에서 필터링하여 제공한다.
```

<br>

### 게시판 페이지
```
목적
- 현재 게시판의 게시글 목록을 제공
- 현재 게시판의 인기글 목록을 제공
- 현재 게시판의 공지글 목록을 제공
- 현재 게시판의 게시글을 대상으로 하는 검색 기능 제공

접근 대상
- 모든 사용자

레이아웃
- Main Layout

필요 데이터
- 현재 게시판의 게시글 목록
- 현재 게시판의 인기글 목록
- 현재 게시판의 공지글 목록

사용자 동작
- 게시판 탐색
  - 현재 게시판의 게시글 / 인기글 / 공지글 조회
- 게시판 검색
  - 현재 게시판의 게시글 / 인기글 / 공지글 대상으로 제목 혹은 내용 기반 검색
- 게시글 상호작용
  - 게시글 상세 이동
  - 게시글 작성 이동 (로그인)

호출 API
- 현재 게시판의 게시글 목록 조회
  - GET /posts?boardId={boardId}&page={pageNumber}
- 현재 게시판의 인기글 목록 조회
  - GET /posts/popular?boardId={boardId}&page={pageNumber}
- 현재 게시판의 공지글 목록 조회
  - GET /notices?boardId={boardId}&page=1
- 현재 게시판의 게시글 검색
  - GET /posts?boardId={boardId}&keyword={keyword}&searchType={TITLE | CONTENT}
- 현재 게시판의 인기글 검색
  - GET /posts/popular?boardId={boardId}&keyword={keyword}&searchType={TITLE | CONTENT}

비고
- 게시글 목록과 인기글 목록은 20개씩 페이지네이션해서 제공한다.
- 공지글은 최근 20개 목록만을 제공한다.
- 모든 글은 최신순으로 제공한다.
```

### 게시글 페이지
```
목적
- 게시글의 세부 페이지를 제공
- 게시글 작성자의 게시글 수정 페이지 이동 기능 제공
- 게시글 작성자의 게시글 삭제 기능 제공
- 게시글의 댓글 조회 기능 제공
- 댓글 작성자의 댓글 수정/삭제 기능 제공
- 게시글 추천/비추천 기능 제공
- 댓글 추천/비추천 기능 제공

접근 대상
- 모든 사용자

레이아웃
- Main Layout

필요 데이터
- 현재 게시글 정보
- 현재 게시글 작성자 정보
- 현재 게시글의 댓글 목록
- 사용자의 현재 게시글 추천/비추천 정보 (로그인)
- 사용자의 댓글 추천/비추천 정보 (로그인)

사용자 동작
- 게시글
  - 게시글 조회
  - 게시글 작성자의 프로필 페이지 이동
  - 게시글 수정 페이지 이동 (로그인, 내 글)
  - 게시글 삭제 (로그인, 내 글)
  - 게시글 추천/비추천 지정 및 취소
- 댓글 및 대댓글
  - 댓글 목록 조회
  - 대댓글 목록 조회
  - 댓글 및 대댓글 작성자의 프로필 페이지 이동
  - 댓글 및 대댓글 작성 (로그인)
  - 댓글 및 대댓글 수정 (로그인, 내 댓글)
  - 댓글 및 대댓글 삭제 (로그인, 내 댓글)
  - 댓글 및 대댓글 추천/비추천 지정 및 취소

호출 API
- 게시글 조회
  - GET /posts/{postId}
- 게시글 삭제
  - DELETE /posts/{postId}
- 게시글 추천/비추천
  - POST /posts/{postId}/reactions
- 게시글 추천/비추천 취소
  - DELETE /posts/{postId}/reactions
- 댓글 목록 조회
  - GET /posts/{postId}/comments?page={pageNumber}
- 대댓글 목록 조회
  - GET /comments/{commentsId}/replies?page={pageNumber}
- 댓글 작성
  - POST /posts/{postId}/comments
  - parentCommentId = null
- 대댓글 작성
  - POST /posts/{postId}/comments
  - parentCommentId != null
- 댓글 및 대댓글 수정
  - PATCH /comments/{commentId}
- 댓글 및 대댓글 삭제
  - DELETE /comments/{commentId}
- 댓글 및 대댓글 추천/비추천
  - POST /comments/{commentId}/reactions
- 댓글 및 대댓글 추천/비추천 취소
  - DELETE /comments/{commentId}/reactions


비고
- 추천과 비추천은 동시에 지정될 수 없다.
- 현재 사용자의 게시글과 댓글, 대댓글 추천 여부를 화면에 표시한다.
- 댓글과 대댓글은 각각 10개씩 페이지네이션하여 제공된다.
- 대댓글은 사용자가 '대댓글 조회' 버튼을 선택할 때마다 다음 페이지를 추가로 조회하여 제공한다.
- 대댓글은 1단계까지만 허용한다.
- 작성자 이름을 클릭했을 때, 작성자가 본인이라면 내 프로필 페이지로 이동하고, 타인이라면 타인의 프로필 페이지로 이동한다.
```
